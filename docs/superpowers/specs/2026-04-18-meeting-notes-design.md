# AI 会议纪要工具 — UI 设计规范

## 概述

Electron 桌面应用，捕获系统音频，实时转写 + AI 摘要，本地处理，零配置开箱即用。

---

## 技术栈

- **框架**：Electron（跨平台，本地音频捕获）
- **音频**：ffmpeg（内嵌）捕获系统音频，BlackHole / WASAPI 虚拟声卡路由
- **ASR**：Riva 流式识别（自托管）或 飞书/腾讯会议 API
- **说话人分离**：pyannote.audio
- **AI 摘要**：通义千问 / Kimi（本地 API Key，用户自填）
- **前端**：React + TailwindCSS

---

## 页面结构

3 个核心页面，左侧导航切换，单窗口设计。

| 页面 | 路径 | 说明 |
|------|------|------|
| 录制页 | `/` | 默认首页，录制控制 |
| 纪要页 | `/notes/:id` | 单次会议详细纪要 |
| 历史页 | `/history` | 所有会议列表 |

## 录制页（首页）

### 空闲状态

- 大型麦克风图标居中（灰色）
- 设备状态提示：当前设备 + 是否有声源
- 两个按钮：
  - **▶ 开始录制**（蓝色主按钮，全宽）
  - **🎵 切换设备**（白底，次级按钮）

### 录制中状态

- 顶部：红色圆点脉冲动画 + 计时器 `HH:MM:SS`
- 状态提示：说话人检测数量 + 段落数
- **实时预览区**：流式转写文字，边说边显示
  - 每条带时间戳 + 说话人标签
  - 新文字淡入，视觉上有节奏感
- 自动保存提示条（每 30 秒保存一次）
- 两个按钮：
  - **⏹ 停止录制**（红色主按钮）
  - **📋 全屏纪要**（白底）

### 异常状态（断线 / 无声）

- 告警边框（红色）
- 显示：中断位置 + 最后正常时间
- 两个按钮：
  - **🔄 重新检测设备**（蓝色主按钮）
  - **🎵 切换设备**（白底）

## 纪要页

### 页面结构

1. **顶部信息栏**：会议标题 + 时长 + 说话人数 + 日期
2. **AI 摘要区**：蓝底，左边框高亮，行动项加粗
3. **说话人标签栏**：颜色标签（蓝/绿/黄），点击可编辑姓名
4. **转写内容区**：按时间顺序，每段带说话人颜色 + 时间戳
5. **底部操作栏**：三按钮

### 交互细节

- 点击说话人标签 → 行内编辑姓名，Enter 保存
- 点击转写文字 → 修正错字，自动同步摘要
- 复制全文 → Markdown 格式，包含时间戳和说话人
- AI 摘要：录制中显示"摘要生成中…"，录制完成后一次性呈现

## 设计决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 技术栈 | Electron | 隐私优先，数据不离开本机 |
| 录制预览 | 实时预览（方案 A） | 用户感知工作状态，降低焦虑 |
| 摘要时机 | 录制完成后一次性生成 | 流式摘要增加复杂度，MVP 先不做 |
| 导航 | 左侧固定导航 | 3 个页面，切换简单 |
